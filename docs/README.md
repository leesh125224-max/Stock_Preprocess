# 특징주 데이터 추출

## 1. 개요
주식 카페 '나는 주식트레이더다'의 특징주 정리본(디선 리포트)을 활용해 제미나이(Gemini) API 분석용 데이터를 만드는 시스템.
단순 수집이 아니라 AI가 시장 흐름을 잘 학습할 수 있도록 고품질의 통계 데이터를 확보하는 것이 최종 목적.

## 2. 주요 기능
1.  **HWP 자동 변환**: `win32com` 라이브러리를 통해 HWP 파일을 자동으로 제어하여 CSV로 변환.
2.  **스마트 날짜 인식**: 파일이 위치한 폴더명(`25년`, `26년`)을 자동으로 인식하여 데이터에 연도 반영.
3.  **전역 그룹화 (Global Aggregation)**: 모든 파일의 데이터를 통합하여, '이유(테마)'가 동일한 항목들을 하나로 묶어 통계 산출.
4.  **데이터 정제 (Cleaning)**:
    *   오타 보정 (예: 2982% -> 29.82%)
    *   비정상 수치(30% 초과) 필터링
    *   결측치(빈 등락률) 자동 채움 기능
    *   불필요 데이터 제외 ("상장 첫날", 빈 이유 등)

## 3. 프로젝트 구조

```text
특징주/
├── execution/
│   └── 특징주.py      # 메인 실행 코드 (HWP 변환 -> 파싱 -> 정제 -> 통합)
├── data/
│   ├── 특징주/        # 원본 HWP 파일 저장소 (연도별 폴더 구분)
│   │   ├── 25년/
│   │   └── 26년/
│   ├── 특징주_csv/    # 변환된 중간 CSV 파일들이 저장되는 곳
│   └── db/
│       └── 특징주.csv # [최종 결과물] AI 학습용 통합 데이터
└── docs/
    └── README.md           # 개발 기록
```

## 4. 개발 내용 및 의사결정 (Dev Log)

### 소스 데이터 선정 (Source Selection)
*   초기 네이버 뉴스 API 활용 뉴스 크롤링 시도.
    *   뉴스 데이터의 비정형성으로 상승 사유 및 등락률 정량화에 한계 확인.
*   '나는 주식트레이더다' 카페 '디선' 리포트 확인.
    *   당일 상승 종목/등락률/이유가 명확히 정리되어 있어 데이터로써 가치 높음.
    *   뉴스 크롤링 중단 및 디선 리포트 활용 전용 모듈(`특징주.py`) 개발 진행.

### 데이터 파싱 전략 (Parsing Strategy)
*   HWP 표(Table) 직접 파싱 시도.
    *   셀 병합 등 구조적 문제로 데이터 누락 발생 확인.
*   HWP 자체 기능인 `SaveAs("CSV")` 활용, 텍스트(CSV)로 1차 변환 후 파싱하는 우회 전략 채택.
    *   복잡한 문서 구조 내에서도 데이터 누락 없이 100% 추출 성공.

### 제미나이(Gemini) API 최적화
*   수집 데이터 활용 시 토큰 제한(분당 25만) 및 AI 환각 현상(과거 종목 기억 의존) 우려.
*   날짜 및 종목명 컬럼 제거 결정.
    *   오직 '이유(Theme)'와 '등락률' 정보만 남겨 데이터 구조 경량화.
    *   AI가 '종목'이 아닌 '재료의 크기'에만 집중하도록 유도하여 분석 정확도 향상 기대.

### 전역 그룹화 (Global Aggregation)
*   초기 파일(날짜) 단위 수집 진행하였으나 시계열 통계 및 거시적 테마 분석 어려움.
*   전체 기간 데이터를 하나로 통합하는 전역 그룹화 로직 적용.
    *   특정 테마 발생 시 시장 반응(등락률) 통계 확보 가능.

### 휴먼 에러 보정 (Human Error Correction)
사람 작성 리포트 특성상 발생하는 실수 시스템적 보완.
*   **등락률 누락**: 작성자 편의상 2등주 등락률 생략(빈칸) 패턴 확인.
    *   내림차순 정렬 특성 이용하여 이전 순위 값 상속(Forward Fill) 처리.
*   **오타 보정**: `2982%` 등 30% 초과 이상치 발견.
    *   자릿수 분석 로직으로 정상 소수점(`29.82%`) 자동 변환 구현하여 데이터 신뢰성 확보.
